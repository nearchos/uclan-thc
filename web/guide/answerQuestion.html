API Guide: answerQuestion

Usage

Submits a proposed answer for the current question. If the question is location-sensitive, you need to use the updateLocation API call first to update the location. A correct answer is rewarded with +10 points.

URL format:
http://uclan-thc.appspot.com/api/csv/answerQuestion?answer=...&session=...

Parameters

  • session: the session ID, typically retrieved after calling startQuiz
  • answer: the proposed answer

Possible outcomes

  • When: Answer is correct & More questions available
    Status: "OK", Message: ""
    Followed by: "correct,unfinished"
    Example: 
        OK,
    correct,unfinished
  • When: Answer is correct & No more questions available (finished all questions in category)
    Status: "OK", Message: ""
    Followed by: "correct,finished"
    Example: 
        OK,
    correct,finished
  • When: Answer is incorrect
    Status: "OK", Message: ""
    Followed by: "incorrect"
    Example: 
        OK,
    incorrect
  • When: Answer is correct BUT the location is unknown or incorrect
    Status: "OK", Message: ""
    Followed by: "unknown or incorrect location"
    Example: 
        OK,
    unknown or incorrect location
  • When: One or more parameters are missing
    Status: "Invalid or missing parameters", Message: "Valid parameter list 'answerQuestion?answer=...&session=...'"
    Example: 
        Invalid or missing parameters,Valid parameter list \
                          'answerQuestion?answer=...&session=...'
  • When: The session is unknown
    Status: "Unknown session", Message: "The specified session ID could not be found"
    Example: 
        Unknown session,The specified session ID could not be found

Example

Issuing this URL 
http://uclan-thc.appspot.com/api/csv/answerQuestion?answer=2\
                    &session=agtzfnVjbGFuLXRoY3IUCxIHU2Vzc2lvbhiAgICAgMOVCgw

returns 
OK,
correct,unfinished
